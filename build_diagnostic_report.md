## 4. Root Cause Analysis

### Why the certification script fails locally

The certification script (`scripts/certify_installer_inputs.py`) is hardcoded to check for `dist/LabReceiptSystem.exe`. On Linux, PyInstaller produces `dist/LabReceiptSystem` (no `.exe` extension). This causes 2 failures:

1. **EXE not found** — checks for `.exe` which doesn't exist on Linux
2. **Source path does not resolve** — ISS `Source: "..\..\dist\LabReceiptSystem.exe"` resolves to non-existent path on Linux

**On GitHub Actions (windows-latest), both checks pass** because PyInstaller produces `.exe` on Windows.

### Why the CI job was failing (original issue)

The original CI failure was caused by the `rstrip("*")` bug in the certification script that made wildcard path validation check the wrong directory. This was fixed in the previous task.
